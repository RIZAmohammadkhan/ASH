Ratatui Powered TUI

Scrollable history, searchable model pickers, and inline spinners all wrapped in a lightning-fast Rust terminal interface.

Smart Retry Loop

If a command fails (like missing permissions or packages), ash feeds the stderr back to the model to fix it automatically—up to 3 times.

Dynamic OpenRouter Models

Fetches up-to-date models directly from OpenRouter API. Supports free-tier only mode, or plug in your key for full access.

Minimal Context Injection

Automatically shares your OS, shell, PATH tool hints, and working directory to ensure accurate commands without bloated tokens.

Download & Install

Download the standalone executable for your operating system. No dependencies required.

↓ Windows (.exe) ↓ macOS (.tar) ↓ Linux (.tar)

1. Configure your API Key

Ash creates a configuration file automatically on first run. Add your OpenRouter API key to unlock all models.

# ~/.config/ash/config.toml
openrouter_api_key = "sk-or-v1-..."
default_model = "openrouter/auto"
shell = "/bin/zsh"

2. Run Initial Query

Open your terminal and start using ash immediately.

ash "find the 20 largest files here"

Advanced: Compile from Source

git clone https://github.com/rizamohammadkhan/ash.git
cd ash
cargo install --path .
